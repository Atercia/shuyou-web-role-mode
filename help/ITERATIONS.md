# 迭代记录总览

## 迭代哲学

本项目采用**渐进式原型开发**方法，每个迭代版本都产生一个可运行的原型，逐步累积功能。

### v0.3.0 - 记忆碎片系统 💎
**日期**: 2026-04-14  
**目标**: 添加核心游戏机制

#### 主要工作
1. 设计记忆碎片数据结构
2. 实现碎片渲染（水晶效果）
3. 添加悬浮标签显示
4. 实现碰撞检测系统
5. 创建交互确认弹窗

#### 游戏设计
- **碎片属性**: 名字、颜色、大小、用时、危险等级
- **颜色等级**: 红(安全) → 青(低危) → 蓝(中等) → 绿(高危) → 金(极危)
- **大小影响**: 越大用时越长
- **随机生成**: 8-12个碎片随机分布

#### 技术亮点
- 使用 Sprite 实现文字标签
- 发光材质和光晕效果
- 组件间事件通信
- Teleport 实现弹窗挂载

#### 交付物
- 记忆碎片系统
- 交互弹窗组件
- 完整的游戏体验

## 开发心得

### 成功经验
1. **渐进式开发**: 每个版本都有可运行的成果
2. **组件化设计**: 逻辑清晰，易于维护
3. **类型安全**: TypeScript 减少运行时错误

### 待改进
1. 需要添加单元测试
2. 性能优化（LOD、对象池）

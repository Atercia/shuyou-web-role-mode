# 迭代记录总览

## 迭代哲学

本项目采用**渐进式原型开发**方法，每个迭代版本都产生一个可运行的原型，逐步累积功能。

### v0.3.0 - 记忆碎片系统 💎
**日期**: 2026-04-14  
**目标**: 添加核心游戏机制

#### 主要工作
1. 设计记忆碎片数据结构
2. 实现碎片渲染（水晶效果）
3. 添加悬浮标签显示
4. 实现碰撞检测系统
5. 创建交互确认弹窗

#### 游戏设计
- **碎片属性**: 名字、颜色、大小、用时、危险等级
- **颜色等级**: 红(安全) → 青(低危) → 蓝(中等) → 绿(高危) → 金(极危)
- **大小影响**: 越大用时越长
- **随机生成**: 8-12个碎片随机分布

#### 技术亮点
- 使用 Sprite 实现文字标签
- 发光材质和光晕效果
- 组件间事件通信
- Teleport 实现弹窗挂载

#### 交付物

### v0.4.0 - 碎片内部场景与哲学交互系统 🌀
**日期**: 2026-04-14  
**目标**: 实现碎片内部的情境广场和哲学对话系统

#### 主要工作
1. 创建情境广场场景（神秘紫色调）
2. 实现三门系统（回忆之门、挑战之门、秘密之门）
3. 创建三个哲学NPC（存在之影、时间旅者、真理守门人）
4. 实现哲学对话系统（存在主义、时间哲学、认识论）
5. 使用Pinia重构状态管理（替代Vue emit）
6. 添加NPC交互圆圈和虚幻效果

#### 游戏设计
- **三门系统**: 不同颜色传送门通向不同场景
- **哲学情境**: 每个NPC对应一个哲学领域的深度思考
- **选择机制**: A/S/D三个选择对应不同哲学立场
- **交互限制**: 每个NPC只能交互一次，交互后变虚幻

#### 技术亮点
- Pinia全局状态管理
- 交互检测优化（最近对象优先）
- MeshBasicMaterial发光效果
- NPC待机动画和虚幻材质

#### 交付物

### v0.5.0 - 隐喻碎片系统与背包功能 🎒
**日期**: 2026-04-14  
**目标**: 完善隐喻碎片机制，实现角色背包系统

#### 主要工作
1. 设计隐喻碎片稀有度系统（普通/稀有/史诗/传说）
2. 实现完整度机制（碎片→残缺→较完整→完整）
3. 创建奖励弹窗（显示稀有度星级和完整度进度条）
4. 实现背包系统（Inventory Store + InventoryPanel）
5. 添加碎片生成算法（概率分布）
6. 背包UI设计（左侧滑出面板，按稀有度排序）

#### 游戏设计
- **稀有度概率**: 普通50% / 稀有25% / 史诗15% / 传说10%
- **完整度范围**: 30-100%，分四个等级
- **背包容量**: 20格，按稀有度自动排序
- **视觉反馈**: 彩色边框标识稀有度，进度条显示完整度

#### 技术亮点
- TypeScript枚举类型定义
- 计算属性实现自动排序
- 动态样式绑定（稀有度颜色）
- 空格键确认关闭弹窗

#### 交付物
- 隐喻碎片类型定义
- Inventory Store状态管理
- RewardModal奖励弹窗
- InventoryPanel背包面板

## 开发心得

### 成功经验
1. **渐进式开发**: 每个版本都有可运行的成果
2. **组件化设计**: 逻辑清晰，易于维护
3. **类型安全**: TypeScript 减少运行时错误

### 待改进
1. 需要添加单元测试
2. 性能优化（LOD、对象池）
3. 移动端适配
